# 🌟 دليل مشروع WeeklyOS الشامل (Master Guide)

أهلاً بيك في الدليل الشامل لمشروع **WeeklyOS**! الملف ده بيجمع لك كل كبيرة وصغيرة عن النظام، معموليته، الإمكانيات اللي فيه، وازاي كل جزء بيكلم التاني. لو عايز تفهم المشروع من طقطق لسلام عليكم، الملف ده هو دليلك الأساسي.

---

## 🎯 1. فكرة المشروع الأساسية
**WeeklyOS** هو تطبيق ويب متكامل معمول عشان يدير وقتك ومهامك الأسبوعية بذكاء، ومدموج فيه قدرات الذكاء الاصطناعي (Gemini API) عشان يساعدك توزع مهامك، تفضي دماغك (Brain Dump)، وتقيّم أداءك أسبوعياً. التطبيق مصمم يكون "Production Ready" بنسبة 100% وبسرعة عالية.

---

## 🛠️ 2. التركيبة التكنولوجية (Tech Stack)
المشروع مبني بأحدث التقنيات عشان يضمن سرعة وأداء وخبرة مستخدم (UX) ممتازة:

- **الواجهة الأمامية (Frontend):** React.js باستخدام Vite و TypeScript.
- **تنسيق واجهة المستخدم (Styling):** Tailwind CSS (مدمج مع Material 3 Colors وإمكانيات الـ Dark Mode).
- **إدارة الحالة (State Management):** Zustand (ومربوطة بالـ localStorage عشان الداتا تفضل موجودة).
- **الخلفية (Backend) وقاعدة البيانات:** Supabase (قاعدة بيانات PostgreSQL، نظام تصديق Auth، و Real-time Updates).
- **وظائف السيرفر (Edge Functions):** `ai-handler` عبر Supabase للرد على طلبات الذكاء الاصطناعي بطريقة آمنة.
- **الذكاء الاصطناعي:** Gemini API (من Google) عشان يعمل Schedule للمهام، يكتب تقييمات، ويقدم اقتراحات.

---

## 📱 3. صفحات التطبيق والمميزات (Pages & Features)
المشروع فيه 7 صفحات رئيسية، كل صفحة ليها دور محدد:

1. **SignIn (تسجيل الدخول):** صفحة آمنة بتستخدم Supabase Auth المربوط بالـ `AuthContext` عشان تدخلك للتطبيق بأمان.
2. **Dashboard (اللوحة الرئيسية):** قلب التطبيق، بتعرض لك المهام، الأيام، وتتصل بـ `useWeekStore` عشان تحمل لك أسبوعك الحالي بمهامه.
3. **WeeklyDistribution (التوزيع الأسبوعي):** صفحة بتساعدك توزع مهامك بذكاء وبطريقة مرئية.
4. **FocusedDay (يوم التركيز):** مخصصة لليوم اللي انت شغال فيه دلوقت، وفيها "مؤقت بومودورو" (Pomodoro Timer) شغال تمام.
5. **BrainDump (تفريغ الدماغ):** مكان سريع ترمي فيه كل الأفكار اللي بتيجي على دماغك عشان متروحش منك.
6. **WeeklyEvaluation (التقييم الأسبوعي):** نهاية الأسبوع بتشوف تقييم ذكي من الـ AI لأدائك.
7. **Settings (الإعدادات):** صفحة اعدادات شاملة ومربوطة بـ `useSettingsStore`.

---

## 🗂️ 4. إدارة الحالة (Zustand Stores)
التطبيق شغال على 6 Stores متبنية على `Zustand`، وكلهم بيكلموا قعدة البيانات (Supabase) وكمان بيعملوا حفظ محلي:
1. `useWeekStore`: بيدير الأسابيع والمهام جواها.
2. `useSettingsStore`: لإعدادات المستخدم والتطبيق (Dark mode, timer settings, etc).
3. `useBrainDumpStore`: بيدير الأفكار العشوائية اللي لسه متوزعتش.
4. `usePinnedTasksStore`: للمهام المتثبتة أو المتكررة (Recurrent Logic).
5. `useLayoutStore`: بيدير شكل الصفحة (Sidebars, Modals وغيرها).
6. **AuthContext**: ده Context خاص بالـ Authentication بيظبط الـ JWT token وحالة المستخدم.

---

## 🗄️ 5. قاعدة البيانات (Database Schema)
استخدمنا **7 جداول** في Postgres (عبر Supabase) وكلهم محميين بـ **RLS (Row Level Security)** عشان مفيش مستخدم يشوف داتا غيره:
- **`weeks`**: بيسجل الأسابيع لكل مستخدم.
- **`tasks`**: المهام المرتبطة بالأسبوع.
- **`brain_dump`**: الأفكار الحرة.
- **`pinned_tasks`**: المهام الثابتة اللي بتتكرر.
- **`user_settings`**: الإعدادات المفضلة للمستخدم.
- **`ai_settings` & `ai_keys`**: إعدادات ومفاتيح الـ AI بطريقة آمنة ومتشفرة.

> **💡 أداء عالي:** كل الجداول معمولها Database Indexes مخصصة عشان الـ Data تيجي في طلقة (مثال: `idx_tasks_week_id`). كمان الـ Real-time شغال يعني لو فيه تحديث بيظهر فوراً في الـ UI.

## 🤖 6. الذكاء الاصطناعي والـ Backend (ai-handler Edge Function)
الذكاء الاصطناعي معزول تماماً ومحمي عشان مفيش API Key يظهر للـ Client.
- الـ Frontend بيكلم الـ **`useApi.ts` hook**.
- الـ hook بيبعت Request لـ **Supabase Edge Function** اللي اسمها `ai-handler`.
- جوا الـ `ai-handler`: بيتم التأكد من الـ JWT Token، وبيجهز الـ Prompts ويتواصل مع **Gemini API**.
- بيدعم **5 نهايات (Endpoints)** ذكية: `chat` (دردشة عادية)، `reflection` (تفكير وتأمل)، `challenge` (تحدي أسبوعي)، `insight` (نظرة عميقة)، `schedule` (توزيع المهام أوتوماتيك).

---

## 🔄 7. مسارات البيانات وحركتها (Data Flows)

أمثلة على دورة حياة البيانات جوا المشروع:
- **تسجيل الدخول:** `SignIn Page` ➜ `Supabase Auth` (بيدي الـ Token) ➜ `AuthContext` بيخزنه ➜ يوديك الـ `Dashboard` ويشغل `useWeekStore.initialize()`.
- **اضافة مهمة:** المستخدم بيدوس اضافة ➜ بيكلم `useWeekStore.createTask()` ➜ Store بيكلم `supabase.from('tasks').insert` ➜ الـ DB بترد بالنجاح ➜ الـ UI بيشوف التحديث أوتوماتيك بسبب الـ Real-time subscription.
- **جدولة ذكية للـ Brain Dump:** دوسة زرار ➜ بتبعت الأفكار للـ `ai-handler` (بـ `schedule` endpoint) ➜ الـ AI بيرجع JSON متنسق ➜ الـ Frontend بيفهم الـ JSON ويحوله لمهام في أيام وتتسمح من شاشة الـ Brain Dump.

## 🧪 8. الاختبار والتأمين (Testing & Quality)
المشروع ده متداس عليه شغل وتأمين زي الكتاب ما بيقول:
- 🚀 **100% اختبارات ناجحة:** فيه **245 اختبار** (Tests)، كلها ناجحة وبتغطي الـ Stores، والـ API، و Supabase.
- 🔒 **أمان عالي:** الـ Row Level Security شغال بصرامة، مفيش داتا بتتسرب، والـ Environment Variables متأمنة.
- ⚡ **أداء ممتاز:** عاملين Lazy Loading للصفحات.

## 🛠️ 9. تشغيل المشروع بسرعة (Quick Start)

عشان تبدأ تطوير أو تكمل شغل:
1. **نزل الباكدچز:**
   ```bash
   npm install
   ```
2. **شغل المشروع في بيئة التطوير:**
   ```bash
   npm run dev
   ```
3. **لو احتجت تتأكد من سلامة الاختبارات (Tests):**
   ```bash
   npm test
   ```
